# WorkHub UI 设计规范（生成 UI 的唯一依据）

> 本文档是「让其他网页 / AI 工具生成一致 UI」的标准依据。
> 任何新页面、新组件，必须遵守这里定义的设计变量、布局、组件规格、状态与文案规范。
> 文档与 `ROADMAP.md`（产品需求）、`commands/feature.md`（开发约束）保持一致。

---

## 0. 使用方式（如何用本文档生成 UI）

1. 复制文末「§13 生成 UI 提示词模板」，填入要生成的页面/组件，连同本文档一起发给生成工具。
2. 生成结果必须满足：
   - 只使用 §2 设计变量（颜色 / 字号 / 间距 / 圆角 / 阴影），不得自造色值。
   - 复用 §5 组件库，不重复造轮子。
   - 文案全中文，遵守 §9 文案规范。
   - 键盘交互遵守 §8（与 `ROADMAP.md` §3 一致）。
3. 技术栈固定：**Vue 3 (`<script setup lang="ts">`) + Tailwind CSS v4**，桌面端 Tauri 窗口。

---

## 1. 设计原则

| 原则 | 说明 |
| --- | --- |
| 键盘优先 | 所有核心操作可纯键盘完成；鼠标是补充而非必需。 |
| 搜索驱动 | 入口即搜索，结果即操作，减少层级跳转。 |
| 即时反馈 | 任何打开/复制/保存动作都有中文 Toast 反馈。 |
| 轻量克制 | 浅色底、圆角卡片、留白充足，不堆砌装饰。 |
| 状态可见 | 选中态、加载态、空态、异常态都有明确视觉。 |
| 一致性 | 同类元素全局统一，不一页一个样。 |

## 2. 设计变量（Design Tokens）

> 所有颜色用语义命名，禁止在页面里写裸十六进制。Tailwind v4 在 `style.css` 中用 `@theme` 注册后即可用类名（如 `bg-surface`、`text-primary`）。

### 2.1 颜色

| 语义 token | 值 | 用途 |
| --- | --- | --- |
| `--color-bg` | `#F5F6F8` | 应用底色（最底层背景） |
| `--color-surface` | `#FFFFFF` | 卡片 / 面板 / 列表项背景 |
| `--color-surface-hover` | `#F0F2F5` | 列表项悬停背景 |
| `--color-surface-active` | `#E8F0FE` | 列表项选中背景（键盘高亮） |
| `--color-border` | `#E5E7EB` | 分隔线 / 卡片描边 |
| `--color-text` | `#1F2329` | 主文字 |
| `--color-text-secondary` | `#646A73` | 次要文字 / 描述 |
| `--color-text-placeholder` | `#9AA0A6` | 占位符 / 禁用文字 |
| `--color-primary` | `#3370FF` | 主色（主按钮、链接、聚焦环、高亮命中） |
| `--color-primary-hover` | `#245BDB` | 主色悬停 |
| `--color-success` | `#2BA471` | 成功（已复制、已保存 Toast） |
| `--color-warning` | `#D99100` | 警告（未保存提示） |
| `--color-danger` | `#D54941` | 危险（删除、恢复、异常） |
| `--color-danger-hover` | `#B23A33` | 危险悬停 |

深色主题（`§10` 设置中心可切换「跟随系统/浅色/深色」）：底色翻转为 `#1B1C1E`，surface 为 `#26282B`，文字主色 `#E8EAED`，主色保持 `#3370FF` 提亮到 `#4D82FF`。生成深色页面时只替换底/面/文字三组，不改语义色相。

### 2.2 字体与字号

- 界面字体：`system-ui, "PingFang SC", "Microsoft YaHei", sans-serif`
- 代码字体（片段/命令/路径）：`"JetBrains Mono", "Cascadia Code", Consolas, monospace`

| token | 字号 / 行高 | 用途 |
| --- | --- | --- |
| `text-page-title` | 18px / 26px，600 | 页面标题 |
| `text-section` | 15px / 22px，600 | 区块小标题 |
| `text-body` | 14px / 22px，400 | 正文 / 列表主文字 |
| `text-caption` | 12px / 18px，400 | 描述 / 时间 / 标签 |
| `text-mono` | 13px / 20px | 代码、命令、路径 |

### 2.3 间距（4px 基准）

`space-1=4 / space-2=8 / space-3=12 / space-4=16 / space-5=20 / space-6=24`
- 卡片内边距：16px（`p-4`）
- 列表项垂直内边距：10–12px
- 区块之间：16–24px

### 2.4 圆角

| token | 值 | 用途 |
| --- | --- | --- |
| `radius-sm` | 6px | 标签 / 徽标 / 小输入 |
| `radius-md` | 10px | 按钮 / 输入框 / 列表项 |
| `radius-lg` | 14px | 卡片 / 面板 |
| `radius-xl` | 18px | 弹窗 / 模态 |

### 2.5 阴影

| token | 值 | 用途 |
| --- | --- | --- |
| `shadow-card` | `0 1px 2px rgba(0,0,0,.04), 0 2px 8px rgba(0,0,0,.04)` | 卡片 |
| `shadow-pop` | `0 8px 24px rgba(0,0,0,.12)` | 下拉 / 弹窗 / Toast |
| `ring-focus` | `0 0 0 2px rgba(51,112,255,.35)` | 聚焦环（输入、可聚焦项） |

### 2.6 动效

- 时长：120–160ms；缓动：`ease-out`。
- 仅对 `background-color / opacity / transform` 做过渡，避免布局抖动。
- 弹窗进入：`opacity 0→1` + `translateY(8px→0)`；Toast：淡入 2.5s 后淡出。

## 3. 应用布局（900×600 固定窗口）

```
┌──────────────────────────────────────────────────────┐
│  顶部区 TopBar（常驻搜索框 / 返回 + 页内搜索）   高 56  │
├───────┬──────────────────────────────────────────────┤
│ 左侧  │                                              │
│ 导航  │              主区 MainArea                    │
│ 宽 64 │           （当前页面内容，可滚动）             │
│ (可选)│                                              │
├───────┴──────────────────────────────────────────────┤
│  底部状态栏 StatusBar（快捷键提示 + Toast 锚点） 高 32 │
└──────────────────────────────────────────────────────┘
```

- 窗口固定 900×600，主区是唯一滚动容器，TopBar / StatusBar 固定不滚。
- 左侧导航 64px：图标 + 文字竖排，对应 `Ctrl+1~5`（首页/项目/片段/收藏/设置），选中项左侧 3px 主色条 + `surface-active` 底。
- 主区内容最大宽度不限，左右留 24px 安全边距。

---

## 4. 栅格与密度

- 主区采用单列纵向流式布局为主；表单弹窗内用「label 顶对齐 + 控件占满」单列。
- 列表为信息主体：每项一行卡片式，高度约 56–64px，行间距 8px。
- 信息密度偏「紧凑」（桌面工具属性），但点击热区不小于 32px 高。

## 5. 组件库（V1 核心组件）

> 对应 `ROADMAP.md` §15。每个新增组件需在组件目录建同名 `README.md`，写清 Props / Emits / 示例 / 快捷键。下面给出统一规格。

### 5.1 SearchBox 搜索框
- 外观：高 40px，`radius-md`，`surface` 底，左内置放大镜图标，右侧可选清除按钮。
- 聚焦：显示 `ring-focus`。
- Props：`modelValue`、`placeholder`（默认「搜索项目、文件、代码片段、收藏…」）、`autofocus`。
- 行为：输入防抖 100ms 触发 `search`；`Esc` 清空（已空则交还全局处理）。

### 5.2 ResultList / ResultItem 结果列表项
- 结构：`[类型图标] 标题（命中高亮）  · 标签 · 相对时间    [收藏标记]`
- 状态：
  - 默认：`surface` 底。
  - 悬停：`surface-hover`。
  - 键盘选中：`surface-active` + 左侧 3px 主色条。
- 命中高亮：命中字符用 `text-primary` + 600。
- 单行省略：标题/描述超出用 `…`。
- 主操作随类型变化（见 §8.3）。

### 5.3 EmptyState 空状态
- 居中：插画/图标（48px，弱化）+ 一行中文说明 + 可选主按钮。
- 文案见 §9.3。

### 5.4 TagInput 标签输入
- 回车添加，标签 chip 可点 `×` 删除，单标签 ≤ 20 字。
- chip：`radius-sm`，`surface-hover` 底，`text-caption`。

### 5.5 FormModal 表单弹窗
- 容器：`radius-xl`，`shadow-pop`，宽 480px，半透明遮罩 `rgba(0,0,0,.32)`。
- 结构：标题栏（标题 + 关闭）→ 表单区（label 顶对齐）→ 底部操作（右对齐：次按钮「取消」+ 主按钮「保存」）。
- 校验：失焦/提交校验，错误在字段下方 `text-danger` `text-caption`。
- 快捷键：`Esc` 关闭，`Ctrl+Enter` 提交。

### 5.6 ConfirmDialog 确认弹窗
- 用于删除 / 导入 / 恢复等危险操作（二次确认）。
- 危险操作主按钮用 `danger` 色；文案明确风险（见 §9.4）。

### 5.7 Toast / StatusBar 反馈
- Toast 锚定底部状态栏上方居中，`shadow-pop`，2.5s 自动消失。
- 类型：成功（`success` 圆点）/ 普通 / 错误（`danger` 圆点）。
- 文案：「已复制」「已用 VSCode 打开」「已打开链接」「命令已复制」等。

### 5.8 CodeViewer 代码查看
- 等宽字体、显示行号、浅色语法高亮（仅展示不执行）。
- 右上角「复制」按钮，复制后 Toast「代码已复制」。

### 5.9 HotkeyRecorder 快捷键录制
- 点击进入录制态，捕获组合键并显示；冲突时提示「该快捷键已被占用」。

### 5.10 按钮规范
| 类型 | 样式 |
| --- | --- |
| 主按钮 | `primary` 底 + 白字，`radius-md`，高 36px，悬停 `primary-hover`。 |
| 次按钮 | `surface` 底 + `border` 描边 + 主文字色，悬停 `surface-hover`。 |
| 危险按钮 | `danger` 底 + 白字，用于确认删除/恢复。 |
| 文本按钮 | 无底，`primary` 字，用于「查看全部」等轻操作。 |
| 图标按钮 | 32×32，`radius-md`，悬停 `surface-hover`。 |

### 5.11 标签 / 徽标
- 标签 chip：见 §5.4。列表最多展示 3 个，超出显示 `+N`。
- 分类徽标（片段）：`radius-sm`，弱底色，分类中文显示（见 ROADMAP §6.3）。

## 6. 图标规范

- 线性图标，描边 1.5px，尺寸 16 / 20 / 24 三档。
- 类型图标统一区分：项目、文件、文件夹、代码片段、链接/网址、文档。
- 颜色：默认 `text-secondary`，选中/主操作时 `primary`。

## 7. 页面模板（基于 ROADMAP 页面清单）

> 路由与职责见 `ROADMAP.md` §2.1。下面是各页布局骨架，生成时套用。

- 首页 `/`：TopBar 常驻搜索框 → 无输入时「最近使用」分组列表（10 条）；有输入时「项目/片段/收藏/文件」分组结果（每组≤5，含「查看全部」）。
- 项目列表 `/projects`：页内搜索 + 右上「新增项目」→ ResultItem 列表（名称/描述/标签/更新时间/收藏）。
- 项目详情 `/projects/:id`：四个卡片区块（基础信息 / 常用链接 / 常用命令 / 项目文档）+ 顶部「快速启动」主按钮。
- 片段列表 `/snippets`：搜索 + 分类下拉 + 标签多选 → 列表（标题/分类徽标/标签/代码预览）。
- 片段详情 `/snippets/:id`：只读查看 ↔ 编辑切换 + CodeViewer。
- 文件收藏 `/favorites`：搜索 + 「新增收藏」→ 列表（类型图标/标题/路径或网址）。
- 设置中心 `/settings`：分区（通用 / 快捷键 / 数据管理），表单式控件 + 危险操作二次确认。

每页统一具备：可访问的空状态、加载态骨架、异常态提示（§11）。

## 8. 键盘交互（与 ROADMAP §3 一致，UI 必须可视化提示）

### 8.1 全局
- `Ctrl+Shift+K` 唤起窗口并聚焦搜索框；`Esc` 清空搜索/关弹窗，无可关闭项则隐藏到托盘；`Ctrl+1~5` 切换模块；`Ctrl+,` 打开设置。

### 8.2 列表内
- `↑/↓` 移动选中（到边界停住）；`Enter` 主操作；`Ctrl+Enter` 详情；`Ctrl+C` 复制；`Ctrl+D` 收藏/取消；`Delete` 删除（二次确认）。

### 8.3 Enter 主操作随类型
| 项类型 | Enter 行为 | Toast |
| --- | --- | --- |
| 项目 | VSCode 打开 `code <path>` | 已用 VSCode 打开 |
| 代码片段 / 常用命令 | 复制到剪贴板 | 代码已复制 / 命令已复制 |
| 常用链接 / 网址收藏 | 默认浏览器打开 | 已打开链接 |
| 文件 / 文件夹收藏 | 系统默认程序 / 资源管理器打开 | 已打开 |
| 项目文档 | 系统默认程序打开 | 已打开 |

> UI 提示：底部状态栏实时展示当前页可用快捷键（如「↑↓ 选择 · Enter 打开 · Ctrl+C 复制」）。

## 9. 文案规范（全中文）

### 9.1 总则（对齐 feature.md）
- 所有面向用户的静态文本必须为简体中文，禁止出现英文按钮/提示/标题/占位符。
- 表达自然、符合中国用户习惯，不机翻；按钮用动词短语（如「新增项目」「保存」「删除」）。
- 代码内变量名/函数名/文件名仍用英文。

### 9.2 占位符示例
- 搜索框：搜索项目、文件、代码片段、收藏…
- 项目名称：请输入项目名称；项目描述：一句话描述这个项目；本地路径：选择或粘贴项目目录。
- 标签输入：回车添加标签。

### 9.3 空状态文案
- 首页最近：还没有最近记录，开始搜索或添加项目吧。
- 项目列表：还没有项目，点击右上角"新增项目"创建第一个。
- 常用链接：暂无常用链接。常用命令：暂无常用命令。
- 文件收藏：还没有收藏，添加常用文档、目录或链接吧。

### 9.4 异常 / 确认文案
- 路径/文件不存在：文件不存在或已被移动（提供「重新选择/移除」）。
- 未装 VSCode：未检测到 VSCode，请先安装或配置 code 命令。
- 快捷键冲突：该快捷键已被占用。
- 退出未保存：有未保存的修改，确定退出吗？
- 删除/导入/恢复：一律二次确认，文案明确风险。

## 10. 状态规范（每个可交互元素都要覆盖）

- 默认 / 悬停 / 聚焦 / 选中 / 禁用 / 加载 / 错误。
- 加载：列表用骨架屏（灰块占位），按钮用内联 loading 并禁用。
- 禁用：降透明度至 0.5，`cursor-not-allowed`，不响应交互。
- 选中（键盘高亮）：`surface-active` + 左侧主色条，保证 `↑↓` 可见。

---

## 11. 边界与异常（对齐 ROADMAP §14）

| 场景 | UI 处理 |
| --- | --- |
| 路径/文件不存在 | 中文提示 + 「重新选择/移除」 |
| 未安装 VSCode | 提示安装或配置 code |
| 快捷键冲突 | 录制时检测并提示 |
| 删除操作 | 一律二次确认 |
| 导入/恢复数据 | 覆盖前强确认 + 风险说明 |
| 空数据 | 各页专属中文空状态 |
| 长文本 | 列表单行省略，详情完整展示 |

---

## 12. 无障碍与细节

- 所有可聚焦元素有可见聚焦环（`ring-focus`），不得 `outline:none` 后不补样式。
- 文字对比度满足可读（正文 ≥ 4.5:1）。
- 交互元素提供语义化 `aria-label`（中文）。
- 不依赖颜色单独传达信息（危险态同时有文案/图标）。
