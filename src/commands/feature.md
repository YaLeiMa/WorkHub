# 功能开发命令

用于在当前项目里继续实现新功能时参考。

## 系统语言

【语言规范】

1. 所有面向用户的静态文本必须使用中文（简体中文）
2. 严禁在 UI 中出现英文（包括按钮、提示语、标题、占位符等）
3. 代码中的变量名、函数名、文件名仍使用英文（遵循常规开发规范）
4. 所有中文文案要符合中国用户习惯，表达自然，不要机翻

## 开发原则

基于 `.plan` 中已确认的页面职责推进开发，优先补齐已有流程，不先扩散新模块。

## 开发顺序

1. 先确认改动对应哪个页面或组件。
2. 检查是否已经有 store、路由参数或已有交互可复用。
3. 优先在现有页面闭环内补齐能力。
4. 改动后至少自检页面跳转、状态变更、数据持久化是否仍成立。

## 代码约束
- 无需再引入vue中的ref, reactive等。
- 遵守当前项目的 `vue3` 、 `tailwindcss`、`tauri` 、`TypeScript` 规范。
- 页面样式延续当前圆角卡片、浅色底、移动端优先的视觉风格。
- 不要在没有实际需求时引入新的全局状态或新依赖。
- 新增组件要在组件目录下创建组件文档，描述组件的功能和接收参数的含义。
- 复杂逻辑代码需要写上中文注释。
## 完成标准

- 页面可访问
- 交互链路闭环
- 状态可追踪
- 文案与字段一致
- 不破坏已有演示能力
